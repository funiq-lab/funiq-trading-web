### 1. Stock buy

`POST` /trade/buy

Buy stocks by specifying a total amount. The system will automatically calculate the purchasable shares based on current market price (rounded to trading lot sizes as required).

#### Request headers

Name | Type | Required | Description
---|---|---|---
Content-Type | string | yes | **Fixed value**: "application/json"

#### Request body

Name | Type | Required | Description
---|---|---|---
login_id | string | yes | Login account ID
security_code | string | yes | Security code, 6-digit format<br />**Example**: "000001"
amount | double | yes | Buy amount (unit: 10,000 CNY).<br />**Example**: 1.5 means 15,000 CNY
chasing_rise_halfway | boolean | no | Whether to enable "chasing rise halfway" mode; default: false

#### Example request

```bash
curl --location --request POST 'http://localhost:8080/trade/buy' \
--header 'Content-Type: application/json' \
--data-raw '{
	"login_id": "123456",
	"security_code": "000001",
	"amount": 1.5
}'
```

#### Response

##### Response fields

Name | Type | Description
---|---|---
code | int | Error code, 0 means success
msg | string | Response message
data | object | Order response data
request_id | int64 | Request ID
order_id | int64 | Order ID
exchange_order_id | string | Exchange order ID
batch_id | int64 | Batch ID
login_id | string | Login account ID
security_code | string | Security code
market | int | Market type
side | int | Buy/sell side
order_type | int | Order type
price | double | Order price
quantity | int64 | Order quantity
amount | double | Order amount
status | int | Order status
status_msg | string | Status message
order_time | int64 | Order time (microseconds)
timestamp | int64 | Timestamp (microseconds)

##### Response example

```json
{
	"code": 0,
	"msg": "success",
	"data": {
		"request_id": 1001,
		"order_id": 20001,
		"exchange_order_id": "E123456789",
		"batch_id": 0,
		"login_id": "123456",
		"security_code": "000001",
		"market": 1,
		"side": 0,
		"order_type": 0,
```

### 6. TWAP sell

`POST` /trade/twap-sell

Use TWAP (time-weighted average price) algorithm to split a large sell order into multiple smaller orders evenly over a specified time window.

#### Request headers

Name | Type | Required | Description
---|---|---|---
Content-Type | string | yes | **Fixed value**: "application/json"

#### Request body

Name | Type | Required | Description
---|---|---|---
login_id | string | yes | Login account ID
security_code | string | yes | Security code, 6-digit format
quantity | int64 | yes | Total quantity to sell
start_time | string | no | Start time (ISO8601, e.g. 2024-01-15T09:30:00). If omitted, start immediately
end_time | string | no | End time (ISO8601, e.g. 2024-01-15T09:40:00). If omitted, defaults to start + 10 minutes
interval_seconds | int | no | Interval between sub-orders (seconds), default 10s. Range: 1-3600
enable_price_threshold | bool | no | Enable price threshold protection, default false
price_threshold_ratio | double | no | Price threshold ratio (0.0-1.0), default 0.05 (5%). Pauses when price deviates beyond this ratio

#### Example request

```bash
curl --location --request POST 'http://localhost:8080/trade/twap-sell' \
--header 'Content-Type: application/json' \
--data-raw '{
	"login_id": "123456",
	"security_code": "000001",
	"quantity": 10000,
	"start_time": "2024-01-15T09:30:00",
	"end_time": "2024-01-15T09:40:00",
	"interval_seconds": 10,
	"enable_price_threshold": true,
	"price_threshold_ratio": 0.05
}'
```

#### Response

##### Response fields

Name | Type | Description
---|---|---
code | int | Error code, 0 means success
msg | string | Response message
data | object | TWAP task response
task_id | int64 | Task ID (numeric)
task_id_formatted | string | Task ID (formatted)
login_id | string | Login account ID
security_code | string | Security code
market | int | Market type
total_quantity | int64 | Total sell quantity
interval_seconds | int | Interval between sub-orders (seconds)
enable_price_threshold | bool | Whether price threshold protection is enabled
price_threshold_ratio | double | Price threshold ratio
task_details | object | Task details (status, progress, etc.)

##### Response example

```json
{
	"code": 0,
	"msg": "success",
	"data": {
		"task_id": 1699261700001,
		"task_id_formatted": "TWAP-20240115-001",
		"login_id": "123456",
		"security_code": "000001",
		"market": 1,
		"total_quantity": 10000,
		"interval_seconds": 10,
		"enable_price_threshold": true,
		"price_threshold_ratio": 0.05,
		"task_details": {
			"status": "running",
			"progress": 0.0,
			"executed_quantity": 0,
			"remaining_quantity": 10000,
			"avg_price": 0.0,
			"create_time": 1699261700000,
			"start_time": 1699262600000,
			"end_time": 1699263200000
		}
	}
}
```
